# 关键卡片 — 需求文档

## 项目概述

关键卡片是一个静态网页生成器工具，用于生成静态网页，可直接部署在静态服务器上，比如 GitHub Pages。
能解决知识总结、时间线串联、关注点对照等问题，以网页形式（不同模板）展示，加深记忆。

## 目标用户

任何想要学习提升的人。

使用场景：
1. 读了一本书，提炼为 20 张关键卡片，将卡片数据录入后台后，点击生成网页，选择笔记模板，快速生成网页并预览。
2. 从零碎视频中了解到 DeepSeek 的发展，整理为 10 张关键卡片，录入系统后，点击生成网页，选择时间线模板，快速生成网页并预览。

## 工作流程

```
录入卡片 → 管理模板 → 选择卡片 + 选择模板 → 预览 → 导出静态网页 → 部署（GitHub Pages / Nginx）
```

## 功能需求

- **卡片管理**
  - 卡片 CRUD（标题、正文、标签、时间戳）
  - 卡片搜索/筛选
  - 卡片数据导入/导出（JSON / SQLite）
- **模板管理**
  - 模板 API 规范：输入卡片数据 → 输出完整 HTML
  - 内置 2–3 个默认模板（笔记、时间线、对照）
- **网页生成**
  - 选择卡片子集（按标签筛选 / 手动勾选）+ 选择模板
  - 实时预览
  - 导出为独立目录（HTML + CSS + JS），可直接部署

## 非功能需求

- **离线可用** — 卡片管理和网页生成均无需联网
- **数据可移植** — 卡片数据可导出为标准格式（JSON），迁移到其他实例
- **生成性能** — 1000 张卡片以内生成时间 < 5 秒
- **模板隔离** — 模板间不共享状态，保证可独立扩展

## 技术约束

- 语言：Java21
- 框架：springboot3 + Vue3 + Ant 
- 数据库：SQLite3
- 部署方式：GitHub Pages、Nginx 等静态服务器
