# Feature Specification: 用户注册与登录

**Created**: 2026-07-03  

**Status**: Draft  
**Input**: 用户描述: "请给系统加上用户注册和登陆"

## Overview

为关键卡片系统添加用户认证功能。采用单用户/管理员模式，使用用户名+密码认证，未登录用户强制跳转至登录页，登录后方可使用所有功能。

## User Scenarios & Testing

### User Story 1 - 管理员注册 (Priority: P1)

首次使用系统时，管理员通过注册页面设置用户名和密码，完成初始账户创建。注册成功后自动登录并跳转至卡片管理页面。

**Why this priority**: 没有账户就无法使用系统，注册是所有后续操作的前提。

**Independent Test**: 打开系统，看到注册页面，输入用户名密码提交，成功跳转至主页。

**Acceptance Scenarios**:

1. **Given** 系统中无任何用户账户, **When** 访问系统, **Then** 自动跳转至注册页面
2. **Given** 注册页面, **When** 输入用户名（2-20字符）和密码（6位以上）并提交, **Then** 注册成功，自动登录，跳转至卡片管理页
3. **Given** 注册页面, **When** 用户名或密码不符合要求, **Then** 显示对应校验提示，不提交
4. **Given** 系统中已有用户账户, **When** 访问系统, **Then** 跳转至登录页面而非注册页

---

### User Story 2 - 管理员登录 (Priority: P1)

管理员通过登录页面输入用户名和密码，验证通过后进入系统主界面。

**Why this priority**: 登录是日常使用的入口，与注册同为最高优先级。

**Independent Test**: 已有账户时，访问系统跳转至登录页，输入正确凭据后进入主页。

**Acceptance Scenarios**:

1. **Given** 已注册账户且未登录, **When** 访问系统任意页面, **Then** 强制跳转至登录页
2. **Given** 登录页面, **When** 输入正确的用户名和密码, **Then** 登录成功，跳转至卡片管理页
3. **Given** 登录页面, **When** 输入错误的用户名或密码, **Then** 提示"用户名或密码错误"
4. **Given** 已登录状态, **When** 访问登录页或注册页, **Then** 自动跳转至卡片管理页

---

### User Story 3 - 管理员退出登录 (Priority: P2)

管理员点击退出按钮，清除登录状态，返回登录页面。

**Why this priority**: 完成会话管理闭环，安全性需要。

**Independent Test**: 登录后点击退出，验证返回登录页且无法再访问功能页面。

**Acceptance Scenarios**:

1. **Given** 已登录状态, **When** 点击退出按钮, **Then** 清除登录状态，跳转至登录页
2. **Given** 已退出登录, **When** 尝试访问功能页面, **Then** 被重定向至登录页

---

### Edge Cases

- 系统已有用户时，注册页面不可访问（或隐藏注册入口）
- 密码在数据库中必须加密存储，不可明文保存
- 浏览器关闭后重新打开，登录状态是否保持？（默认：保持，使用持久化 token）
- 并发请求时 token 过期的处理

## Requirements

### Functional Requirements

- **FR-001**: 系统 MUST 提供注册页面，允许设置用户名（2-20字符）和密码（6位以上）
- **FR-002**: 系统 MUST 在无用户账户时自动导向注册页，有账户时导向登录页
- **FR-003**: 系统 MUST 提供登录页面，验证用户名和密码
- **FR-004**: 系统 MUST 在未登录状态下拦截所有功能页面，重定向至登录页
- **FR-005**: 系统 MUST 在登录后显示退出按钮
- **FR-006**: 系统 MUST 退出登录时清除会话状态并重定向至登录页
- **FR-007**: 系统 MUST 对密码进行加密存储
- **FR-008**: 系统 MUST 在已登录状态下阻止访问登录/注册页
- **FR-009**: 系统 MUST 仅允许一个管理员账户存在（单用户模式）
- **FR-010**: 系统 MUST 提供登录状态持久化（浏览器关闭后重新打开无需重新登录）

### Key Entities

- **User**: 管理员账户，属性包括用户名、加密后的密码、创建时间。单用户模式仅一条记录。

## Success Criteria

### Measurable Outcomes

- **SC-001**: 用户可在 30 秒内完成注册或登录操作
- **SC-002**: 未登录用户无法访问任何功能页面，100% 被拦截
- **SC-003**: 密码不以明文形式存储在数据库中
- **SC-004**: 登录状态在浏览器关闭后仍可保持 24 小时以上

## Assumptions

- 单用户/管理员模式：仅一个账户，无需多用户数据隔离
- 用户名+密码认证，不涉及第三方 OAuth 或邮箱验证
- 使用 Session 或 JWT 进行会话管理，本项目采用 JWT（更适合前后端分离架构）
- 离线可用约束不变，认证逻辑完全本地化
- 前端路由守卫 + 后端 API 拦截双重保护
